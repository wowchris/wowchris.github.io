docker安装和命令

发表于 2017-01-17 | 分类于 docker

docker定义

先看下docker的logo

这个图标很形象：
鲸鱼代表了docker这个应用程序，上面的集装箱则代表了docker所承载的内容，每个箱子都是一个容器container，里面装的可以是系统镜像，也可以是应用程序，那么整体上看就是一个docker里可以应用很多个service.

容器式的虚拟机，虚拟机式的容器

查了查网上的各种说法，仍然说不出个所以然，后来在dockerone上看到一篇文章，感觉和我所理解的的很贴近，关于容器与虚拟机的差异，具有普遍共识的特征归纳起来大致有以下几点：

  • 容器实例与主机共享操作系统内核，通过内核提供的运行时隔离能力为服务提供独立的用户域、文件系统、网络以及进程运行空间。虚拟机的每个实例自带操作系统，因而是一种硬件级的虚拟化隔离。
  • 容器通常是专用于运行特定服务的，它的镜像通常只包含运行该服务所需的上下文内容，许多广泛使用的镜像都只有几十MB，甚至几MB大小。虚拟机则需要提供包括内核在内的通用进程运行环境，它的镜像偏向于大而完整的全功能集合，即使一个最小的精简镜像的体积也有几百MB。
  • 容器的使用方式倾向于开箱即用，镜像提供的是一个『不可变的基础设施环境』。虚拟机则倾向于让用户根据所用的系统，自定义初始化操作，使用Ansible、Puppet这样的配置工具来进行基础设施的管理。
  • 容器在启动速度和运行性能上更有优势，虚拟机在安全性上更有优势。

通过这个介绍可以大致说明了虚拟机和容器的区别，类似于apache和nginx的讨论，实际应用看你怎么选了。

安装

 安装官方的说明文档，需要内核版本3.8以上，所以先需要升级内核
我使用是是centos6.5，其他系统可参照官方说明安装https://docs.docker.com/engine/installation/linux/

内核升级

rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org
rpm -ivh http://www.elrepo.org/elrepo-release-6-5.el6.elrepo.noarch.rpm
yum -y --enablerepo=elrepo-kernel install kernel-lt

编辑grub.conf文件，修改引导区

vi /etc/grub.conf

将default=1修改为default=0

最后，通过reboot命令重启操作系统 

[root@localhost ~]# uname -r  
3.10.104-1.el6.elrepo.x86_64

docker.io 和 docker-engine

软件是一样的，区别是:

docker.io is maintained by Ubuntu
docker-engine is maintained by Docker

cnetos6 and centos7

centos6
按照wiki的说法，需要安装EPEL才行，这样安装的docker是1.5版本，docker官方对centos6的支持停留在1.7.1版本，所以不要无谓的升级版本，升级了你反而可能不能用了，出现各种各样的问题，所以在安装docker之前先确定你的需求。
centos7
官网上有详细的安装教程，因为centos7拥有运行docker的高版本内核

centos6安装

rpm -Uvh http://download.fedoraproject.org/pub/epel/6/i386/epel-release-6-8.noarch.rpm  
yum -y install docker-io

centos7安装

准备工作：
yum install device-mapper-event-libs device-mapper device-mapper-event device-mapper-libs 

sudo tee /etc/yum.repos.d/docker.repo <<-'EOF'
[dockerrepo]
name=Docker Repository
baseurl=https://yum.dockerproject.org/repo/main/centos/$releasever/
enabled=1
gpgcheck=1
gpgkey=https://yum.dockerproject.org/gpg
EOF
注意：此方法也适用于centos6版本，且版本为1.7.1
$releasever需要改成对应的版本号，6或者7，不然会提示源错误

命令行 

yum install docker-engine
service docker start

docker info
docker run  hello-world

如果看不到信息那可能是相关软件没有更新，执行yum update吧

获取最新镜像

docker run

本地镜像：
docker images
启动容器
  准备建立个jenkins应用 

docker run -i -t -v /usr/local/src/soft/:/opt/software/ 镜像ID或仓库名称 /bin/bash

  命令解释：  
  -i,"交互模式"启动；  
  -t，启动后进入命令行模式；  
  -v，前面为主机目录，后面为容器目录；  
  /bin/bash,进入命令行

  之后你可以当这个容器是虚拟机，jenkins的安装过程这里不做描述，当配置工作完成后进行保存操作：
  docker commit <container> [repo:tag]    #repo:tag可选  

  最终运行：  
  docker run -d -t -v /usr/local/src/vote_prod:/opt/software -p 8080:3000 --name vote_prod deb-node1.0 bash -c "source /root/.profile; /usr/local/bin/forever /opt/software/vote/bin/www"
强调一下，如果想进入当前运行的容器：
该命令在退出当前容器后不退出后台容器
 
docker exec -it <新容器ID> /bin/bash
